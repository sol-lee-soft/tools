Media Engineering Manual

System Architectures, Storage Behaviors, and Module Operations

Data Persistence Rules

The Media Engineering tools run inside a secure local sandbox engine known as LocalStorage.

When is your data saved?

  • Instant auto-save: Every single text keystroke, checkbox toggle, or file added is saved to browser memory the exact millisecond you change it.
  • Session Resilience: You can refresh the page, close the tab, close your browser, or reboot your machine entirely without losing a single word.

When will your data be erased?

Because the data lives within your web browser's local sandbox, it will only be erased under three conditions:

  1. You intentionally click the Wipe All button inside a specific module.
  2. You explicitly clear your web browser's cookies and site data storage history.
  3. You run CCleaner or automated system cleaning utility scripts that force-wipe local browser cache databases.
CRITICAL CAUTION: Do not use these tools inside private windows or "Incognito Mode". Incognito sandboxes destroy all accumulated LocalStorage records instantly the moment you close the browser window.

1. Continuity Configurator

Designed to enforce environmental constants, visual profiles, and generation guardrails across multi-step asset processes.

How to use it:

  1. Input your active target identifier rules into the fields (Project Name, Active Scene Key).
  2. In the Environmental Constants box, lock down persistent variables (e.g., specific camera lenses, lighting constraints, object vectors, colors, or structural rules).
  3. Flip the pipeline toggle switches to specify memory thresholds (VRAM Optimization) or quota limits.
  4. Click Compile Rule Block. This formats your parameters into clean system string markers and copies them straight to your clipboard automatically.

Safety features:

  • Export (JSON): Downloads your exact tracking settings framework into a physical file on your hard drive, serving as an immutable backup.
  • Wipe All: Flash-clears your configurations to start a brand new sequence tracking project.

2. Script Pipeline

A zero-latency, scene-by-scene screenwriting environment and dialogue reference tracker styled in industry-standard courier proportion spacing.

How to use it:

  1. Click + Add New Scene in the left-hand navigation column to instantiate a fresh block.
  2. Click directly inside the header input field to change or customize the scene description name.
  3. Compose or paste your text directly into the green editor block. Capitalize character names and parentheticals to match traditional manuscript layouts.
  4. Switch between distinct acts fluidly by selecting the tracking titles listed in the navigation column.

Safety features:

  • Each individual entry maintains an isolated text container layer. Deleting a scene via the small × icon will only drop that specific block without impacting neighboring assets.
  • Use the Wipe All Scenes button to completely flash the entire text index when pivoting to a totally new project concept.

3. Project Asset Manager

A smart indexer designed to track audio layers, model files, render states, and milestones without local file system security restrictions.

How to use it:

  1. Click the dashed Browse... button to open a system workspace view panel and select a file.
  2. The script will isolate the item name and attempt an automated type guess based on standard extensions (e.g., mapping .mp4 to Video or .wav to Audio).
  3. Manually adjust the structural type or generation pipeline state (Complete, In Progress, Pending) using the drop-down menus.
  4. Click Track Asset to load the record straight into your ledger list layout.

Workspace Shortcuts:

  • Context Paths: The system automatically logs localized directory path indicators based on the target category. Clicking on any gray path text instantly copies it to your clipboard for fast file navigation on your operating system.
